# Vibe Research Pipeline — RAG Submodule Prototype

Markdown-first, AI-maintained, researcher-readable research knowledge base. Not traditional vector DB PDF QA.

## Quick start

### 1. Initialize a project-specific RAG

```
python scripts/rag/rag_init.py --rag-dir RAG --dimensions methods,models,datasets --template-fields "summary,key findings,limitations,links" --vocabulary methods,models
```

This generates `RAG/template.md`, `RAG/vocabulary.md`, `RAG/index.md`, dimension directories under `RAG/summary/<dimension>/`, and the full skeleton. Do not hardcode dimensions; answer per project.

### 2. Import BibTeX

```
python scripts/rag/import_bib.py --bib path/to/export.bib --rag-dir RAG --dry-run
python scripts/rag/import_bib.py --bib path/to/export.bib --rag-dir RAG
```

Parses, normalizes, deduplicates, and appends entries to `RAG/references.bib`. Reports new vs. skipped.

### 3. Sync PDFs (local match)

```
python scripts/rag/sync_pdf.py --rag-dir RAG --pdf-dir path/to/pdfs --dry-run
python scripts/rag/sync_pdf.py --rag-dir RAG --pdf-dir path/to/pdfs
```

Matches local PDF files by citation key, DOI slug, or title slug. Copies valid PDFs into `RAG/reference/pdfs/`.

### 4. Ingest (generate source pages)

```
python scripts/rag/ingest.py --rag-dir RAG
```

Creates `RAG/summary/sources/<entry-key>.md` for each entry in `references.bib` that lacks a source page. This step creates the page skeleton and metadata.

### 5. Summarize source pages

```
python scripts/rag/summarize_sources.py --rag-dir RAG
```

Extracts local PDF text with `pdftotext` when available and fills placeholder `Summary`, `key findings`, `limitations`, and `links` sections. It only replaces placeholder sections, preserving manual edits.

### 6. Update index

```
python scripts/rag/update_index.py --rag-dir RAG
```

Scans source page frontmatter and rebuilds AUTO blocks on dimension pages. Preserves manual prose outside `<!-- AUTO:BEGIN -->` / `<!-- AUTO:END -->`.

### 7. Query

```
python scripts/rag/query.py --rag-dir RAG --query "method name"
```

Markdown-first search across sources, synthesis, and dimension directories.

### 8. Sync from notes

```
python scripts/rag/sync_from_notes.py --rag-dir RAG --notes-dir path/to/notes
```

Extracts `Decision:`, `Consensus:`, `Open question:`, `Failed:`, `Dispute:`, `Finding:` markers and `[@citation-key]` references into `RAG/summary/synthesis/notes-sync.md`.

### 9. Lint

```
python scripts/rag/rag_lint.py --rag-dir RAG
```

Checks duplicate BibTeX keys, missing PDFs, dead links, off-vocabulary tags, orphan source pages, missing dimension pages, and AUTO block integrity.

### 10. Import Zotero RDF zip

```
python scripts/rag/zip_importer.py --zip path/to/zotero-export.zip --rag-dir RAG --dry-run
python scripts/rag/zip_importer.py --zip path/to/zotero-export.zip --rag-dir RAG
```

Finds `.rdf` metadata, parses entries and PDF attachments, copies valid PDFs, skips HTML snapshots, generates BibTeX, deduplicates.

## Run tests

```
python -m unittest discover -s tests -v
```

32 tests covering: BibTeX parsing, dedup, PDF validation, RDF parsing, zip import, lint rules, directory initialization, and end-to-end `init → import-bib → ingest → query → sync-from-notes → lint` workflow.

## Directory layout

```
RAG/
├── SKILL.md                    # Operation entry point
├── index.md                    # Navigation hub
├── template.md                 # Source page template (generated by rag-init)
├── vocabulary.md               # Controlled vocabulary (generated by rag-init)
├── log.md                      # Operation log
├── references.bib              # Shared references manifest
├── reference/
│   ├── pdfs/                   # Raw PDFs
│   └── imports/                # Import artifacts
└── summary/
    ├── sources/                # One page per paper
    ├── synthesis/              # Cross-source synthesis
    └── <dimension>/            # Project-specific dimension pages

.claude/skills/
├── rag-init/SKILL.md           # Initialization protocol
├── rag/SKILL.md                # Daily operations protocol
├── rag-import/SKILL.md         # Import operations protocol
└── rag-maintain/SKILL.md       # Maintenance operations protocol

scripts/rag/                    # Python CLI tools
tests/                          # Unit and integration tests
```
